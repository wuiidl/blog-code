# Reframe 004 from "agent runtime" to "remote coding companion"

**Date:** 2026-04-23
**Status:** Draft

## Goal

Re-scope the existing `004-agent-runtime/` example to match the reframed
companion post (now *I built a product from my phone*, spec at
`thegugenberger/docs/task-specs/2026-04-20_post-004-coding-from-phone.md`).

The old frame sold the example as a security-argument pattern ("the box
is the trust boundary"). The new post is a story about lowering the
threshold to start a coding session — the same EC2 stack, but in service
of a reader wondering *could I have a remote coding buddy I reach from
my phone?* rather than *how do I harden an agent runtime?*

Code underneath barely changes. Framing, README, and a couple of
missing setup pieces do.

## Background

- Original scaffold spec: `docs/task-specs/2026-04-20_scaffold-and-004-agent-runtime.md` (frozen, shipped).
- Original post spec (now reframed): `thegugenberger/docs/task-specs/2026-04-20_post-004-coding-from-phone.md`.
- User is explicit: not building a rival to OpenClaw; framing is "same shape of setup, different agent inside."
- User does not claim security expertise; the hardening defaults in the CDK came from an AI agent. Reflect this honestly in the README.

## Changes

1. **Rename the folder.** `004-agent-runtime/` → `004-remote-coding-companion/`.
2. **Rewrite the folder `README.md`.** Reframe from *here's a hardened-EC2 pattern* to *here's the minimum cloud setup to run Claude Code on a box you own, reachable from the Claude iOS app*. Keep deploy / connect / destroy sections; rewrite the intro and "what this shows" / "what this is not" sections.
3. **Add the post-implied pieces that are currently missing:**
   - A user-data addendum or separate bootstrap script that installs `tmux`, Node, and Claude Code, and starts a named tmux session the agent lives in.
   - SSM connect instructions plus a short, generic "how you reach this from your phone" note (Claude iOS app over SSM — no private-setup leakage).
   - A "what this is *not*" section: not a production platform, not a substitute for OpenClaw, not security-reviewed for shared use. The minimum kit, nothing more.
4. **Edit the Decisions bullets in the folder README** (not the shipped spec):
   - Drop hardening-as-headline framing.
   - Replace the "generic, not a copy of remote-claude" bullet with "minimal recipe, not a hardened production runtime; a reader who wants a serious multi-user setup should look at OpenClaw."
5. **Honest credit line in the README.** One sentence: *the hardening flags here were generated by an AI agent and are sensible defaults, not an authority-reviewed security posture. Regenerate for your own context before trusting them anywhere real.*
6. **No changes to the CDK hardening itself.** Locked-root, IMDSv2, sysctl hardening, disabled legacy services, dnf-automatic all stay. They're sensible defaults. We're changing framing, not substance.

## Deliverables

Against `blog-code` `main`:

- `git mv 004-agent-runtime 004-remote-coding-companion`
- `004-remote-coding-companion/README.md` — rewritten per above.
- `004-remote-coding-companion/bootstrap.sh` (or extended user-data in `lib/*-stack.ts`) — installs tmux, Node, Claude Code, starts the named tmux session.
- Top-level `README.md` — update the example list entry from *agent runtime* to *remote coding companion*, and update the link-back to the new post slug once the post ships.
- Stack class rename if it reads as `AgentRuntimeStack` — to `RemoteCodingCompanionStack` or similar. Match folder.

## Out of scope

- CDK hardening changes. Defaults stay.
- Scheduled start/stop, credential bootstrap, private-subnet variant — still planned as separate future examples per the original scaffold spec's follow-ups list.
- The post itself. That's tracked in `thegugenberger`.

## Verification

- Folder renamed; no dangling references to `004-agent-runtime` in any README or TS file.
- `pnpm install && pnpm run synth` still produces valid CloudFormation from the renamed folder.
- README intro reads as a story-shaped invitation (*you can have this; here's the recipe*), not a thesis (*this is the right pattern*).
- `bootstrap.sh` or the user-data installs `tmux` + Claude Code and leaves a named session running on boot.
- "What this is not" section names OpenClaw as the serious-production reference.
- Honest-credit line about AI-generated hardening is present in the README.

## Decisions

- **New spec, not a rewrite.** The 2026-04-20 scaffold spec is a shipped historical record. Reframe work gets its own dated spec so the diff of *what changed on the reframe* is legible in isolation.
- **Folder name `004-remote-coding-companion`.** Mirrors the post's language. Avoids both "agent runtime" (too platform-y for the new frame) and any leak of `remote-claude` (private repo name).
- **Stack name changes too.** Keeping `AgentRuntimeStack` in the code while the folder and post use *remote coding companion* would be confusing for a reader who forks.
- **No CDK substance changes.** Hardening defaults are fine; the honest-credit line in the README is the right inoculation, not softer defaults.
